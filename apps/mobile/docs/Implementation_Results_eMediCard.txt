Implementation Results

      The implementation of eMediCard: A Mobile-Based Health Card Management System achieved successful deployment of all major system components, demonstrating that digital transformation of health card processing is both feasible and beneficial for the City Health Office of Davao City. This section presents the outcomes of the deployment process, key achievements realized during implementation, adjustments made based on testing and user feedback, and the system's readiness for full-scale operational use.

Deployment Outcomes

      The system deployment proceeded according to the implementation plan, with both mobile and web-based components successfully configured and made accessible to target users. The mobile application was built using Expo Application Services (EAS) Build, generating production-ready Android packages that were tested across multiple device models representing the range of smartphones commonly used by health card applicants. Installation testing confirmed that the application operated correctly on Android versions 8.0 through 13, with consistent performance across devices from various manufacturers including Samsung, Oppo, Vivo, and Xiaomi.

      The Convex database was configured for production use with appropriate schema definitions, indexes on frequently queried fields, and function permissions ensuring secure data access. Initial data migration populated reference tables with job category definitions, document requirements for each classification, and administrator account credentials. Database performance testing under simulated load conditions confirmed that the serverless infrastructure handled concurrent user operations efficiently, with query response times remaining under acceptable thresholds even when multiple administrators and applicants accessed the system simultaneously.

      Clerk authentication integration was successfully established in the production environment, with email verification workflows tested to confirm reliable delivery of verification messages. Configuration of email sender domains and template customization ensured that verification emails maintained professional branding consistent with City Health Office communications while avoiding spam filter issues that could prevent message delivery. Authentication security features including session management, password complexity requirements, and token-based API authorization functioned as designed, providing robust protection for sensitive health information.

      Maya payment gateway integration transitioned from sandbox testing to production credentials, enabling real-time payment processing through the mobile application. Payment workflow testing confirmed that users could initiate payments, redirect to Maya's secure interface, complete transactions using various payment methods, and return to the application with payment status properly updated. The automatic addition of the ten-peso processing fee was validated to ensure correct fare calculations across all payment transactions.

Key Achievements

      Several significant achievements emerged during the implementation phase, demonstrating the system's effectiveness in addressing the challenges of manual health card processing.

      The email verification process proved highly reliable, with verification messages delivered promptly to user email addresses and activation links functioning correctly to grant dashboard access. This two-factor authentication approach effectively prevents fraudulent registrations while ensuring that users provide valid contact information for system notifications. User feedback indicated that the verification process was straightforward and completed easily, even for users with limited technical experience.

      Document upload and validation workflows operated smoothly, with the system correctly accepting valid file formats (PDF, JPEG, PNG) while rejecting incompatible types with clear error messages. File size validation prevented excessively large uploads that could impact storage and performance, while compression algorithms optimized storage efficiency without compromising image quality needed for administrative review. Document preview capabilities in the administrative dashboard allowed CHO staff to efficiently review uploaded files with zoom functionality for examining fine details in medical results and identification documents.

      QR code generation and scanning functionality achieved complete success, with each approved application generating unique, scannable codes that encoded booking information securely. Testing with multiple QR code scanning applications confirmed broad compatibility, and the inspector interface accurately validated codes while displaying attendee information for verification. The dual scan system (check-in and check-out) at orientation venues effectively tracked attendance with precise timestamps, automating orientation completion verification that previously required manual roll call procedures.

      Maya payment integration operated reliably, with test transactions confirming that payment initiation, secure processing, and status callbacks functioned correctly. Users reported that the payment interface was intuitive and that confirmation messages provided clear feedback about transaction success. For manual payment uploads, the receipt image storage and administrative validation workflow allowed CHO staff to efficiently verify payments made at physical locations while maintaining complete audit trails.

      The administrative dashboard successfully consolidated application management functions into efficient workflows. Document verification interfaces allowed rapid review of multiple applications with clear approve/reject options and predefined rejection reason templates that streamlined communication with applicants. Payment validation screens presented transaction details in organized formats that facilitated quick verification. Orientation scheduling tools enabled administrators to create sessions with specified dates, times, venues, and capacities, with automatic availability calculations preventing overbooking. Application review consolidation screens provided comprehensive views of all requirements, enabling informed approval decisions without navigating between multiple interfaces.

Minor Adjustments Made

      Based on testing results and user feedback during the implementation phase, several refinements were made to enhance system usability and performance.

      Interface optimizations for mobile devices addressed feedback regarding button sizes, spacing, and tap targets to ensure comfortable interaction on smaller screens. Text contrast was enhanced in certain screens to improve readability under various lighting conditions, and font sizes were adjusted to maintain legibility without requiring excessive scrolling. These modifications improved the overall user experience, particularly for older applicants who may have visual challenges.

      Page loading speed improvements resulted from optimizing image compression algorithms, implementing progressive loading for document previews in the administrative dashboard, and refining database query patterns to reduce unnecessary data retrieval. These optimizations decreased initial loading times and improved responsiveness when navigating between screens, making the application feel more fluid and reducing user frustration during periods of slower internet connectivity.

      Real-time notification enhancements refined the timing and frequency of push notifications to balance keeping users informed with avoiding notification fatigue. Notifications were consolidated to deliver single messages for multiple related events rather than sending separate alerts for each status change. Notification text was revised to provide clearer, more actionable information, such as specifying which documents require attention rather than generic "application update" messages. These adjustments increased notification effectiveness while reducing the likelihood that users would disable notifications entirely.

      Error message clarity was improved throughout the application based on user confusion identified during testing. Technical error messages were replaced with user-friendly explanations that suggested corrective actions, such as "Please check your internet connection and try again" rather than generic "Network error" messages. Form validation errors were enhanced to highlight specific fields requiring correction and explain validation requirements clearly.

Readiness Statement

      The successful implementation and testing of eMediCard demonstrates that the system is ready for full-scale deployment and operational use within the City Health Office of Davao City. All core functionalities operate reliably, security measures protect sensitive health information appropriately, and user interfaces provide accessible experiences for both applicants and administrators.

      The system effectively addresses the primary challenges motivating its development, including reducing physical congestion at processing centers by enabling remote application submission, minimizing manual processing workload through automated workflows and validation, improving transparency through real-time status tracking and notifications, and enhancing data accuracy by eliminating manual transcription errors. Performance testing confirms that the infrastructure scales appropriately to handle expected user volumes, including peak periods during January when renewal applications surge.

      City Health Office staff who participated in training and testing expressed confidence in their ability to manage the system effectively, noting that administrative workflows aligned well with existing processes while introducing efficiency improvements. Applicant users who completed test applications reported positive experiences, appreciating the convenience of mobile access, clarity of instructions, and transparency of status tracking.

      The foundation established through this implementation provides a robust platform for future enhancements and expanded capabilities. Potential improvements identified for future development include online payment expansion to support additional payment methods beyond Maya, data analytics and reporting tools to provide deeper insights into application patterns and processing efficiency, multi-category support for additional health card types beyond the current Yellow, Green, and Pink classifications, integration with other City Health Office systems for comprehensive health services management, and iOS application development to support Apple device users.

      Overall, the implementation results confirm that eMediCard successfully transforms health card processing from a manual, paper-based system into an efficient, secure, and user-friendly digital platform. The system is prepared to support the health card needs of Davao City's workforce while positioning the City Health Office at the forefront of digital health services innovation in the Philippines.
